## 🎯 Quick Links by Task

### I want to...

**Report a bug:**
1. Read: [Bug report template](.github/ISSUE_TEMPLATE/bug_report.md)
2. Label with: `type: bug`, `priority: *`, component label

**Suggest a feature:**
1. Read: [Feature request template](.github/ISSUE_TEMPLATE/feature_request.md)
2. Label with: `type: feature`, `needs discussion`

**Fix a bug:**
1. Find issue with: `type: bug`, `good first issue`
2. Follow: [ONBOARDING.md](ONBOARDING.md) - Developer path
3. Branch name: `bugfix/issue-description`

**Add a feature:**
1. Check: [MILESTONES.md](MILESTONES.md) for planned features
2. Create issue or find existing
3. Branch name: `feature/feature-description`

**Improve documentation:**
1. Find: `type: documentation` issues
2. Branch name: `docs/doc-description`
3. Quick process - fast to review!

**Update dependencies:**
1. Branch name: `chore/update-dependencies`
2. Test thoroughly
3. Check security advisories

**Fix urgent production issue:**
1. Branch name: `hotfix/urgent-fix`
2. Label: `priority: critical`
3. Fast-track review process

### Search Tips

**GitHub search:**
```
# Find good first issues
is:issue is:open label:"good first issue"

# Find documentation tasks
is:issue is:open label:"type: documentation"

# Find frontend bugs
is:issue is:open label:"type: bug" label:"component: frontend"

# Find help wanted
is:issue is:open label:"help wanted"
```

**Grep in repository:**
```bash
# Find all TODO comments
grep -r "TODO" .

# Find specific function
grep -r "function_name" .

# Find in markdown files
grep -r "search_term" *.md
```
